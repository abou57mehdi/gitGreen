How do I use the Energy Prediction app?

The Energy Prediction app helps you forecast future energy consumption based on historical data and machine learning models. Input your current data, and the app will predict future energy needs and trends.

Access the App:

Navigate to the Energy Prediction page from the home page.

Fill Out the Form:
  • Voltage (V): Enter a value between 0 and 300 volts.
  • Current (A): Enter a value between 0 and 50 amperes.
  • Frequency (Hz): Enter a value between 0 and 60 Hertz.
  • ESP32 Temperature (°C): Enter a value between -40 and 125°C.
Submit the Form:

After filling out the fields, submit the form to receive a prediction.

View Prediction:

The app will display the prediction based on the input values.

Constraints and Considerations:
  • Ensure Data Accuracy: Input values must be within the specified ranges for accurate predictions.
  • Form Validation: The form validates input values. Correct any errors indicated by the form.
  • Model and Scaler: Ensure the model, scaler, and encoder files are correctly placed in the `energy_prediction` directory.
  • Server Logs: For debugging, check server logs if predictions are incorrect or if errors occur.
  • Handling Missing Data: The app will prompt you to correct any missing or invalid data before proceeding with predictions.
How do I use the Anomaly Detection app?

The Anomaly Detection app identifies unusual patterns or anomalies in your data using machine learning models. Upload your data, and the app will analyze it to detect any anomalies.

Access the App:

Navigate to the Anomaly Detection page from the home page.

Upload Data File:
  • Select the file containing your data from your local storage.
  • Ensure the file is in CSV format and contains the necessary columns.
Fill Out the Form:
  • Data File: Upload a CSV file containing the data to be analyzed.
Submit the Form:

After uploading the file, submit the form to start the anomaly detection process.

View Results:

The app will display the results, highlighting any detected anomalies. You can view both the original data with anomaly markings and a filtered list of anomalous entries.

Constraints and Considerations:
  • Ensure Data Accuracy: Your CSV file should include columns such as `energia`, `WORKSTATION_CPU_POWER`, and `GPU_0_UTILIZATION` for accurate anomaly detection.
  • Model and Scaler: Ensure the model and scaler files are correctly placed in the `anomaly_detection` directory.
  • Server Logs: For debugging, check server logs if anomalies are not detected or if errors occur.
  • Handling Missing Data: The app will prompt you to correct any missing or invalid data before proceeding with anomaly detection.
